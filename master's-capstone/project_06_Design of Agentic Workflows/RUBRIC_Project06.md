## Rubric

Use this project rubric to understand and assess the project criteria.

## Technical Implementation

| Criteria | Submission Requirements |
| --- | --- |
| Agent Implementation | A working agent or agentic workflow is implemented using tools and frameworks covered in the Agentic AI course. The system executes without errors and includes agent initialization, reasoning or decision logic, memory or state handling, and at least one form of tool or external interaction. |
| System Architecture | The agent’s architecture is clearly defined and documented. Core components such as persona, reasoning loop, memory, and tools are identifiable, and their interactions are accurately described or diagrammed. |
| Execution and Behavior | The agent is executed on representative inputs or scenarios. Observable outputs, actions, or decisions are shown, demonstrating that the system behaves as designed. |

## Analytical Reasoning

| Criteria | Submission Requirements |
| --- | --- |
| Task Definition and Scope | The agentic task is clearly defined, including the system’s goal, scope, and boundaries. The choice of single-agent or multi-agent design is appropriate for the task and is logically justified. |
| Decision Logic and Design Choices | The student explains how the agent makes decisions and why specific design choices were made. Reasoning is coherent and aligned with the observed system behavior. |
| Evidence and Justification | Claims about system behavior and effectiveness are grounded in the student’s own implementation and execution results. Citations are used appropriately when referencing agentic AI concepts or design patterns. |

## Ethical and Responsible Practice

| Criteria | Submission Requirements |
| --- | --- |
| Safety and Reliability | The system includes at least one safeguard, constraint, or fallback mechanism intended to improve reliability, transparency, or control. The purpose of these measures is clearly explained. |
| Ethical Reflection | The written report identifies at least one ethical concern relevant to the agentic system, such as autonomy, misuse, bias, accountability, or societal impact. The discussion is specific to the student’s system and grounded in its design or behavior. |

## Communication and Presentation

| Criteria | Submission Requirements |
| --- | --- |
| Notebook or Script Organization | The notebook or script is clearly organized and readable. Code, Markdown explanations, and execution outputs are easy to follow and support understanding of the system. |
| Written Report Quality | The Agentic AI System Design Report is well-structured and clearly written. It communicates technical design, behavior, and ethical reasoning in language suitable for both technical and non-technical audiences. |

## Reproducibility and Professional Practice

| Criteria | Submission Requirements |
| --- | --- |
| Reproducibility | A correct requirements.txt file is included and generated from the working environment. Any datasets, configuration files, prompt templates, or external dependencies are clearly documented so the system can be rerun. |
| Academic Integrity | The agent’s behavior and outputs come from the student’s own implementation and executions. Analysis and ethical discussion reference specific observed behaviors and design decisions rather than hypothetical or generic examples. |
